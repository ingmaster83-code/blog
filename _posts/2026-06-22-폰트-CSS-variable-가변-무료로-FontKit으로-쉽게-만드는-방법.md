CSS variable을 활용한 가변 폰트 작업, 매번 CDN 링크만 복사해 쓰다가 커스텀 폰트가 필요할 때면 난감합니다. 직접 woff2로 변환하고, CSS에 @font-face를 선언하고, variable을 매핑하는 과정이 번거롭기 때문입니다. 이번 글에서는 **폰트 CSS variable 가변 무료**로 해결할 수 있는 실용 도구를 소개합니다.

## FontKit: 무료 가변 폰트 CSS variable 생성 도구

[FontKit](https://fontkit.wooahouse.com)은 업로드한 폰트 파일을 분석해 즉시 CSS variable 코드로 변환해주는 온라인 도구입니다. 설치나 회원가입 없이 브라우저에서 바로 사용할 수 있으며, **CSS variable 가변 폰트**를 실무에 바로 적용할 수 있도록 도와줍니다.

### 주요 기능: 한 번에 해결되는 웹폰트 작업

- **자동 CSS variable 생성**: 업로드한 폰트의 굵기, 스타일, 패밀리명을 감지해 `--font-primary`, `--font-weight-bold` 같은 변수로 자동 매핑합니다.
- **실시간 프리뷰**: 생성된 코드를 즉시 미리보기로 확인하며, 폰트가 어떻게 렌더링되는지 테스트할 수 있습니다.
- **가변 폰트 지원**: woff2, ttf 등 다양한 포맷을 지원하며, 가변 축(weight, width 등)이 있는 폰트라면 CSS variable로 바로 활용 가능합니다.
- **클립보드 복사**: 생성된 CSS 코드를 한 번에 복사해 프로젝트에 붙여넣기만 하면 됩니다.

### 실무 활용 팁: FontKit으로 워크플로우 개선하기

#### ### 1. 디자인 시스템에 통합하기
프로젝트의 루트 CSS 파일에 `:root {}` 블록을 만들고, FontKit이 생성한 변수를 그대로 선언하세요. 이후 모든 컴포넌트에서 `font-family: var(--font-primary)`로 일관성 있게 폰트를 관리할 수 있습니다.

#### ### 2. 반응형 가변 폰트 설정
미디어 쿼리 없이도 `--font-size-base` 같은 변수를 활용해 뷰포트에 따라 폰트 크기를 유동적으로 조절할 수 있습니다. FontKit으로 만든 가변 폰트를 사용하면 `font-weight`도 변수로 제어되므로, 반응형 디자인에서 훨씬 간결한 코드를 작성할 수 있습니다.

#### ### 3. 다국어 사이트에서 폰트 관리
여러 언어를 지원하는 사이트라면, 각 언어별 폰트를 별도로 로드하지 않고 FontKit에서 생성한 CSS variable로 통합하세요. `--font-ko`, `--font-en`처럼 변수명만 구분하면 유지보수가 크게 줄어듭니다.

### 웹폰트 최적화를 고려한 변수 활용

- **preload와 조합**: 생성된 CSS variable을 사용할 폰트는 `<link rel="preload">`로 미리 로드해 성능을 높이세요.
- **fallback 설정**: `--font-primary: 'MyFont', sans-serif;`처럼 변수에 fallback을 포함시키면, 폰트가 로드되지 않아도 레이아웃이 깨지지 않습니다.
- **subset 폰트 사용**: 한국어 사이트라면 영문/한글 subset 폰트를 각각 생성해 FontKit에 업로드하면, 파일 크기를 최적화하면서도 CSS variable로 편리하게 관리할 수 있습니다.
