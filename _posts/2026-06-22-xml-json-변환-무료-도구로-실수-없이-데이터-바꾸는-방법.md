### 활용 팁: 변환 전 체크리스트 3가지

아무리 좋은 **xml json 변환 방법**을 사용해도 입력 데이터가 잘못되면 결과가 엉망이 됩니다. WooaDev를 사용할 때 미리 확인하면 좋은 실수 방지 팁을 알려드립니다.

#### 첫째, XML 루트 요소 확인하기
XML은 반드시 하나의 루트 요소를 가져야 합니다. `<root><item>...</item></root>`처럼 최상위 태그가 하나인지 꼭 확인하세요. 루트가 없거나 두 개 이상이면 WooaDev가 오류를 반환합니다. 이때 간단히 루트 태그로 감싸주면 해결됩니다.

#### 둘째, 속성과 텍스트 노드 구분하기
XML에서 `<name lang="ko">홍길동</name>`처럼 속성과 텍스트가 섞여 있는 경우, JSON 변환 시 속성은 `@lang`, 텍스트는 `#text` 같은 특수 키로 변환됩니다. WooaDev는 이 구조를 자동으로 보존해주지만, 결과 JSON을 사용할 때 이 키를 인지하고 파싱해야 합니다. 미리 구조를 예상하고 작업하면 혼란을 줄일 수 있습니다.

#### 셋째, 인코딩 문제에 대비하기
한글이 포함된 XML 파일에서 변환 후 글자가 깨지는 경우가 있습니다. 이는 XML 선언부의 인코딩과 실제 파일 인코딩이 다를 때 발생합니다. WooaDev는 UTF-8을 기본으로 처리하므로, 변환 전에 파일을 UTF-8로 저장하거나 텍스트 에디터에서 인코딩을 통일하는 습관을 들이세요.
