# 노토 산스 한국어 폰트 무료로 사용하는 가장 쉬운 방법

최근 블로그나 웹사이트 디자인을 하면서 "본문에 깔끔한 고딕체를 써야겠다" 싶어서 여러 폰트를 찾아보다가, 가장 무난하면서도 세련된 **노토 산스 한국어 폰트 무료** 버전을 발견했습니다. 그런데 문제는 이 폰트를 직접 다운로드해서 설치하려면 용량도 크고, 웹폰트로 적용하려면 복잡한 설정이 필요하다는 점이었죠. 저처럼 디자인에 문외한인 사람에게는 꽤 번거로운 일이었습니다.

그러다가 우연히 발견한 온라인 도구가 바로 **FontKit**입니다. 이 글에서는 제가 실제로 사용해본 경험을 바탕으로, 노토 산스 한국어 폰트를 무료로 설치 없이 바로 사용하는 방법을 단계별로 알려드리겠습니다.

## FontKit으로 노토 산스 한국어 폰트 무료 변환하기

FontKit([WooaPDF 무료로 사용하기](https://fontkit.wooahouse.com))은 로컬에 설치된 폰트를 웹폰트로 변환해주는 온라인 도구입니다. 특히 노토 산스 한국어 폰트처럼 용량이 큰 폰트를 웹사이트에 최적화된 형태로 가볍게 만들어줍니다. 설치나 회원가입이 전혀 필요 없고, 모든 작업이 브라우저 안에서 완료됩니다.

### 1단계: 노토 산스 한국어 폰트 파일 준비하기

먼저 구글 폰트(Google Fonts)에서 노토 산스 한국어 폰트 파일을 다운로드합니다. 이때 TTF 형식의 파일을 받아야 나중에 변환하기 편리합니다. Noto Sans KR로 검색하면 Regular, Bold, Light 등 여러 굵기를 찾을 수 있는데, 저는 주로 사용할 Regular와 Bold 두 가지만 준비했습니다.

### 2단계: FontKit 웹사이트 접속

웹 브라우저를 열고 [FontKit](https://fontkit.wooahouse.com)에 접속합니다. 메인 화면이 심플해서 처음 봤을 때 "이게 끝인가?" 싶을 정도로 직관적이었습니다. 상단에 파일 업로드 버튼이 보이고, 몇 가지 옵션만 선택하면 됩니다.

### 3단계: 폰트 업로드 및 옵션 설정

업로드 버튼을 클릭해 아까 준비한 노토 산스 한국어 폰트 TTF 파일을 선택합니다. 그러면 자동으로 파일이 분석되면서 폰트 이름과 스타일이 표시됩니다. 여기서 중요한 옵션이 두 가지 있습니다:

- **서브셋(Subset) 설정**: 한국어 폰트는 한글, 영문, 숫자, 특수문자가 포함되어 있어 용량이 큽니다. '한국어 전용'으로 서브셋을 설정하면 불필요한 글리프를 제거해 파일 크기를 70% 이상 줄일 수 있습니다.
- **출력 형식**: WOFF2와 WOFF를 선택할 수 있습니다. 최신 브라우저라면 WOFF2가 더 압축률이 높으니 이걸 추천합니다.

### 4단계: 변환 및 다운로드

옵션 설정이 끝났으면 '변환' 버튼을 클릭합니다. 놀랍게도 작업이 10초도 안 걸렸습니다. 원본 TTF 파일이 3MB였는데, 서브셋 적용 후 WOFF2로 변환하니 500KB로 줄었습니다. 변환이 완료되면 자동으로 zip 파일이 다운로드되며, 안에는 폰트 파일과 함께 CSS 코드도 들어있습니다.

## 실제 웹사이트에 적용하는 방법

변환된 노토 산스 한국어 폰트를 실제 블로그나 웹사이트에 적용하는 것도 간단합니다. 다운로드한 zip 파일을 압축 풀고, 폰트 파일을 웹 서버에 업로드합니다. 그리고 CSS에 아래 코드를 추가하면 끝입니다.

```css
@font-face {
    font-family: 'Noto Sans KR';
    src: url('NotoSansKR-Regular.woff2') format('woff2');
    font-weight: 400;
    font-style: normal;
}
```

이렇게 하면 방문자들이 별도의 폰트 설치 없이도 내가 지정한 노토 산스 한국어 폰트로 웹페이지를 볼 수 있습니다.

## 주의사항: 라이선스와 호환성 체크

노토 산스 한국어 폰트는 SIL 오픈 폰트 라이선스(OFL)를 따르기 때문에 상업적 사용이 자유롭습니다. 하지만 FontKit으로 변환할 때는 몇 가지 주의할 점이 있습니다.

- **파일 형식 확인**: FontKit은 TTF와 OTF 형식만 지원합니다. 다른 형식의 폰트는 먼저 변환해야 합니다.
- **브라우저 호환성**: WOFF2는 모든 브라우저에서 지원되지만, 구형 브라우저(IE 11 이하)에서는 WOFF를 추가로 제공하는 것이 좋습니다. FontKit에서 WOFF도 함께 변환하면 됩니다.
- **개인정보 보호**: 앞서 FAQ에서도 언급했듯이, 모든 처리는 로컬 브라우저에서만 이루어지므로 민감한 폰트 파일도 안전하게 사용할 수 있습니다.
