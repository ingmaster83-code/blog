## vs code node js 실행 방법: 실수 2. 파일 저장 없이 실행하려는 습관

VS Code에서 코드를 작성하고 터미널에 `node app.js`를 입력했는데 예전 버전의 결과만 나온다면, 대부분 **파일을 저장하지 않았기 때문**입니다. VS Code는 자동 저장 기능이 꺼져 있으면 수동으로 저장(Ctrl+S)해야만 최신 코드가 반영됩니다.

**해결책: WooaVS는 자동 저장 + 즉시 실행**
WooaVS는 코드를 입력하는 순간 자동으로 저장되고, 실행 버튼 한 번으로 최신 상태의 코드가 실행됩니다. 터미널 명령어를 외우거나 `node` 명령어를 타이핑할 필요도 없습니다.

3. **코드 작성** 왼쪽 에디터에 원하는 Node.js 코드를 입력합니다.

## vs code node js 실행 방법: 실수 3. 모듈 import 경로 오류

Node.js에서 외부 라이브러리나 다른 파일을 불러올 때 `require('./utils')` 같은 경로를 잘못 입력하면 'Cannot find module' 오류가 발생합니다. 특히 파일 구조가 복잡해질수록 실수할 확률이 높아집니다.

**해결책: WooaVS의 파일 트리로 구조를 한눈에**
WooaVS는 왼쪽 사이드바에 **파일 트리**를 제공합니다. 현재 폴더 구조를 시각적으로 확인하면서 경로를 입력할 수 있어, 상대 경로 실수를 줄여줍니다.
