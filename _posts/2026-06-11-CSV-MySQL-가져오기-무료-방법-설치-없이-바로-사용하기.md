CSV 파일을 MySQL 데이터베이스로 가져오는 작업은 생각보다 자주 삐걱거립니다. 필드 구분자가 맞지 않거나, 인코딩이 깨지거나, Workbench가 대용량 파일에서 툭하면 멈춰버리기 일쑤죠. 이번 글에서는 **CSV MySQL 가져오기 무료 방법**을 찾다가 실제로 겪는 여러 실수와 그 해결책을 중심으로, 별도 설치 없이 바로 사용할 수 있는 온라인 도구를 소개합니다.

## CSV MySQL 가져오기에서 자주 겪는 3가지 실수

### 1. 인코딩 문제로 한글이 깨지는 경우
CSV 파일이 EUC-KR로 저장되어 있거나, MySQL 기본 문자셋과 불일치할 때 발생합니다. Workbench에서 가져오기 마법사를 실행해도 컬럼명이 깨져서 식별이 불가능해집니다.

### 2. 대용량 CSV 업로드 시간 초과
수십만 행 이상의 CSV를 MySQL Workbench로 직접 임포트하면 타임아웃 오류가 발생하거나 프로그램이 멈춥니다. 명령줄로 처리하기에는 번거롭고 실수할 가능성이 높습니다.

### 3. 날짜 및 숫자 형식 불일치
CSV 안의 '2024-01-01' 같은 날짜 데이터가 MySQL의 DATE 형식과 정확히 매칭되지 않아 NULL 값으로 들어가거나, 소수점 구분자(쉼표 vs 점)가 달라 데이터가 왜곡됩니다.

### WooaSheet 사용법 단계별 가이드

#### 1단계: CSV 파일 준비 및 업로드
- CSV 파일을 UTF-8 인코딩으로 저장하세요. 엑셀에서 '다른 이름으로 저장 → CSV UTF-8'을 선택하면 됩니다.
- [WooaSheet](https://wooasheet.wooahouse.com) 사이트에 접속한 후 '파일 선택' 버튼을 눌러 CSV 파일을 업로드합니다.

#### 2단계: 데이터 미리보기 및 설정
- 업로드가 완료되면 CSV의 첫 10행이 미리보기로 표시됩니다. 컬럼명이 정확한지, 데이터가 올바르게 파싱되었는지 확인하세요.
- 첫 행을 컬럼명으로 사용할지, 데이터로 취급할지 선택할 수 있습니다. MySQL 테이블 생성 시 컬럼명으로 활용하려면 '첫 행을 헤더로 사용' 옵션을 켜세요.

#### 3단계: MySQL INSERT 쿼리 생성
- 'SQL 생성' 버튼을 클릭하면 도구가 자동으로 `CREATE TABLE` 문과 `INSERT INTO` 문을 생성합니다.
- 생성된 SQL에는 날짜 형식, 숫자 포맷이 자동으로 맞춰져 있어 앞서 언급한 형식 불일치 문제가 해결됩니다.

#### 4단계: 쿼리 실행 및 데이터 확인
- 생성된 SQL을 복사하여 MySQL Workbench, phpMyAdmin, 또는 명령줄 클라이언트에 붙여넣고 실행하세요.
- 실행 후 `SELECT COUNT(*) FROM 테이블명`으로 CSV의 행 수와 일치하는지 확인하면 데이터 손실 여부를 바로 알 수 있습니다.
