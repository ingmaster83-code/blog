## 기능 상세: WooaDev로 API 응답 테스트하는 방법

### 1. HTTP 메서드 선택과 URL 입력

WooaDev에 접속하면 가장 먼저 보이는 화면에 'GET', 'POST', 'PUT', 'DELETE' 등 주요 HTTP 메서드 버튼이 있습니다. 일반적인 **REST API 테스트**에서 가장 많이 사용하는 GET 방식을 선택하고, 테스트할 API 주소를 입력칸에 붙여넣기 합니다.

예를 들어, 공개 API인 `[https://jsonplaceholder.typicode.com/posts/1`을](https://jsonplaceholder.typicode.com/posts/1`을) 입력해보세요. 이 주소는 가상의 게시글 데이터를 반환하는 테스트용 API입니다.

### 2. 헤더와 바디 설정 (선택 사항)

인증 토큰이 필요한 API라면 '헤더' 탭에서 Authorization 키를 추가할 수 있습니다. POST나 PUT 방식으로 데이터를 보낼 때는 '바디' 탭에서 JSON 형식으로 데이터를 작성합니다. WooaDev는 JSON 자동 완성 기능을 제공해 오타를 줄여줍니다.

### 3. 요청 보내기와 응답 확인

'전송' 버튼을 클릭하면 즉시 **API 응답 확인**이 시작됩니다. 화면 하단에 상태 코드(200, 404, 500 등), 응답 시간, 그리고 응답 본문(Response Body)이 깔끔하게 표시됩니다. JSON 데이터는 자동으로 트리 구조로 접혀서 보여주기 때문에 원하는 데이터를 빠르게 찾을 수 있습니다.
