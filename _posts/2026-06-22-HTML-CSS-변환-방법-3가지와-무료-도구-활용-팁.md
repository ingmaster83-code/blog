## 활용 팁: 자주 겪는 실수와 해결법

### 인라인 스타일을 무조건 외부로 빼지 마세요

많은 초보자가 인라인 스타일을 전부 CSS로 변환하려다가 오히려 코드가 더 지저분해집니다. 중요한 포인트는 **CSS 변환 효율성**입니다. 한 번만 사용되는 스타일은 인라인으로 남겨도 괜찮습니다. WooaDev의 자동 변환 기능을 사용할 때는 "전체 변환"보다는 "선택 변환" 모드를 활용해 꼭 필요한 부분만 CSS로 분리하는 것이 좋습니다.

### 선택자 이름은 의미 있게 지으세요

**HTML CSS 변환 방법**에서 가장 많이 하는 실수 중 하나가 `.box1`, `.box2` 같은 무의미한 클래스명을 사용하는 것입니다. WooaDev는 변환 시 클래스명을 자동 생성하지만, 사용자가 직접 수정할 수 있는 기능도 제공합니다. 예를 들어 헤더 영역의 스타일이라면 `.header-title`처럼 명확하게 지어야 나중에 유지보수가 쉬워집니다.

### 반응형 디자인을 고려한 변환

데스크톱 전용 HTML을 **HTML CSS 변환 방법**으로 처리한 후 모바일에서 디자인이 깨지는 경우가 많습니다. 이를 방지하려면 변환 후에 미디어 쿼리를 추가해야 합니다. WooaDev는 **HTML CSS 변환 도구**이긴 하지만, 변환된 CSS 파일에 반응형 룰을 직접 추가할 수 있는 기능을 제공하지는 않습니다. 따라서 변환 후에는 반드시 `@media` 규칙을 직접 작성해주세요.
