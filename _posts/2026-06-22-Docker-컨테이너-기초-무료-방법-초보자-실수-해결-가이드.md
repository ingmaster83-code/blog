### 왜 WooaDev가 Docker 컨테이너 기초 학습에 적합한가?

#### 1. 설치 과정에서 발생하는 실수 원천 차단
- **문제**: Docker Desktop 설치 시 "Hardware assisted virtualization and data execution protection must be enabled in the BIOS" 오류.
- **해결**: WooaDev는 클라우드 기반이라 BIOS 설정이나 가상화 활성화가 필요 없습니다. 접속 즉시 `docker --version`을 입력해 보세요.

#### 2. 이미지 pull 실패 시 디버깅 연습
- **문제**: `docker pull nginx` 실행 시 "manifest for nginx:latest not found" 같은 태그 오류.
- **방법**: WooaDev 터미널에서 `docker search nginx`로 정확한 이미지 이름을 확인한 후 `docker pull nginx:alpine`으로 시도해보세요. 오류 메시지를 직접 읽고 수정하는 습관이 중요합니다.

#### 3. 포트 충돌 없이 컨테이너 실행
- **문제**: 로컬에서 `docker run -p 80:80 nginx` 실행 시 "port is already allocated" 오류.
- **해결법**: WooaDev는 격리된 환경이므로 포트 충돌이 발생하지 않습니다. 대신 `-p 8080:80` 같은 다른 포트 매핑을 연습하며 `docker ps`로 실행 상태를 확인해보세요.

### 실제 사용 후기: 30분 만에 배운 컨테이너 생명주기

처음 WooaDev를 열었을 때, 저는 그저 "Hello World"만 출력해보려 했습니다. 하지만 도구가 너무 간편해서 자연스럽게 더 많은 실험을 하게 되었습니다.

1. **컨테이너 실행 및 중지**: `docker run -d --name my_nginx nginx`로 백그라운드 실행을 배우고, `docker stop my_nginx`로 중지하는 과정에서 컨테이너의 상태 변화를 실시간으로 확인했습니다.
2. **볼륨 마운트 오류 시뮬레이션**: `docker run -v /host:/data nginx` 같은 잘못된 경로 입력 시 "invalid mount config" 오류를 만나고, 올바른 경로 규칙을 익혔습니다.
3. **네트워크 격리 이해**: `docker network create my_bridge`를 생성하고 두 컨테이너를 연결하는 실습을 진행하며, "컨테이너 간 통신이 안 된다"는 초보자의 고민을 직접 해결했습니다.

이 모든 과정이 설치 없이 무료로 가능하다는 점이 가장 큰 장점입니다.

## 초보자가 반드시 알아야 할 Docker 컨테이너 기초 명령어

**Docker 컨테이너 기초**를 마스터하기 위해 반드시 알아야 할 핵심 명령어를 정리했습니다.

### 컨테이너 생성 및 실행
- `docker run <이미지>`: 이미지를 기반으로 컨테이너 실행
- `docker run -it <이미지> /bin/bash`: 인터랙티브 모드로 진입
- `docker run -d <이미지>`: 백그라운드 실행 (데몬 모드)

### 컨테이너 관리
- `docker ps`: 실행 중인 컨테이너 목록
- `docker ps -a`: 모든 컨테이너 목록 (중지된 것 포함)
- `docker stop <컨테이너ID>`: 컨테이너 중지
- `docker rm <컨테이너ID>`: 컨테이너 삭제

### 이미지 관리
- `docker images`: 로컬에 저장된 이미지 목록
- `docker pull <이미지>`: 이미지 다운로드
- `docker rmi <이미지>`: 이미지 삭제

### 실수 포인트: docker ps와 docker ps -a의 차이를 모르면 컨테이너를 찾지 못합니다
종료된 컨테이너는 `docker ps`에 보이지 않습니다. 반드시 `-a` 옵션을 붙여야 찾을 수 있습니다. WooaDev에서 직접 두 명령어를 번갈아 실행하며 차이를 체감해보세요.
