## 웹사이트 속도를 좌우하는 폰트, 자체 호스팅이 답이다

블로그나 웹사이트를 운영하다 보면 구글 폰트 속도 문제에 부딪힌 적이 있을 겁니다. 페이지가 예쁘게 뜨다가 갑자기 폰트가 깜빡이거나, 아예 텍스트가 보이지 않는 순간이 생기죠. 이는 외부 CDN에서 폰트를 가져오느라 생기는 지연 때문입니다. 그래서 많은 개발자와 블로거가 **구글 폰트 자체 호스팅 무료** 방법을 찾고 있습니다. 직접 서버에 폰트 파일을 올리면 속도도 빨라지고, 구글 서버 장애에도 영향을 받지 않죠.

## FontKit: 구글 폰트 자체 호스팅을 위한 무료 도구

직접 폰트 파일을 다운로드하고 CSS를 작성하는 건 번거롭습니다. 특히 여러 굵기(weight)와 언어를 지원하려면 시간이 많이 걸립니다. 이런 작업을 단 30초로 줄여주는 도구가 바로 [FontKit](https://fontkit.wooahouse.com)입니다.

FontKit은 브라우저에서 바로 작동하는 웹 앱으로, 모든 처리가 로컬에서 이루어집니다. 구글 폰트 자체 호스팅 무료 도구 중에서도 가장 간편하고, 개인정보 걱정이 없어 추천합니다.

### FontKit 사용법 단계별 가이드

#### 1단계: 원하는 구글 폰트 선택
먼저 구글 폰트 사이트에서 사용할 폰트를 고릅니다. 예를 들어 'Noto Sans KR', 'Roboto' 등 자주 쓰는 폰트를 선택하세요. 링크를 복사해두면 FontKit에 붙여넣기만 하면 됩니다.

#### 2단계: FontKit 사이트 접속
[FontKit 무료로 사용하기](https://fontkit.wooahouse.com) 링크를 클릭해 접속합니다. 별도의 회원가입이나 설치가 필요 없습니다.

#### 3단계: 폰트 URL 입력 및 옵션 설정
화면 중앙에 있는 입력창에 복사한 구글 폰트 URL을 붙여넣습니다. 아래 옵션에서 필요한 굵기(예: 400, 700)와 문자셋(예: korean, latin)을 선택합니다. 한국어 블로그라면 'korean'과 'latin'을 모두 체크하세요.

#### 4단계: 다운로드 버튼 클릭
"Download" 버튼을 누르면 잠시 후 ZIP 파일이 생성됩니다. 이 파일에는 다음과 같은 내용이 들어 있습니다:
- 모든 폰트 파일 (woff2, woff 등 최적화된 형식)
- 바로 사용할 수 있는 CSS 파일
- 설치 가이드 텍스트

#### 5단계: 웹사이트에 적용
ZIP 파일을 압축 풀고, 폰트 폴더를 웹사이트 루트 디렉토리에 업로드합니다. CSS 파일의 경로만 자신의 서버 구조에 맞게 수정하면 끝입니다. 사이트 속도가 눈에 띄게 개선되는 걸 체감할 수 있습니다.

## 자체 호스팅 시 주의사항

FontKit이 자동으로 최적화해주지만, 몇 가지 확인할 점이 있습니다.

**파일 용량 관리**: 한글 폰트는 용량이 큽니다. Noto Sans KR 하나만 해도 여러 굵기를 포함하면 수 MB에 달합니다. 필요한 굵기만 선택해 불필요한 용량을 줄이세요.

**라이선스 확인**: 대부분의 구글 폰트는 SIL 오픈 폰트 라이선스로 자체 호스팅이 자유롭습니다. 하지만 상업용 폰트는 별도 라이선스를 확인하세요.

**CSS 경로 수정**: 다운로드한 CSS 파일은 상대 경로로 되어 있습니다. 서버 구조에 맞게 `url()` 경로를 절대 경로나 올바른 상대 경로로 변경해야 합니다.

**캐싱 설정**: 폰트 파일은 자주 바뀌지 않으므로, 서버에서 캐시 기간을 길게 설정해 성능을 더 높일 수 있습니다.

## 구글 폰트 자체 호스팅 무료, 이제 FontKit으로 끝

외부 폰트 의존성을 없애고 사이트 속도를 높이는 가장 확실한 방법은 자체 호스팅입니다. FontKit 덕분에 더 이상 복잡한 명령어나 FTP 작업에 시간을 쏟지 않아도 됩니다. 브라우저만 있으면 누구나 **구글 폰트 자체 호스팅 무료**로 구현할 수 있습니다.

직접 사용해보니 한글 폰트도 깔끔하게 변환되고, CSS까지 자동 생성되어 정말 편리했습니다. 사이트 성능을 고민 중이라면 지금 바로 [FontKit](https://fontkit.wooahouse.com)을 방문해보세요. 속도 최적화의 첫걸음이 시작됩니다.
