맥 터미널 사용법 기초를 익히면 매일 반복되는 파일 정리, 폴더 이동, 앱 실행 같은 작업을 몇 초 만에 끝낼 수 있습니다. 마우스를 수없이 클릭하던 시간을 줄이고, 키보드만으로 모든 걸 제어할 수 있다는 점이 가장 큰 매력이죠. 하지만 처음 터미널을 열면 검은 화면과 커서만 덩그러니 있어 막막하게 느껴질 수 있습니다. 이 글에서는 초보자도 바로 따라 할 수 있는 핵심 명령어와 함께, 생산성을 높이는 구체적인 활용 팁을 소개합니다. 모든 과정은 무료 도구 [WooaMac](https://mactools.wooahouse.com)에서 제공하는 예제를 기반으로 하니, 부담 없이 따라와 보세요.

## 맥 터미널 사용법 기초: 핵심 명령어 3가지

터미널을 처음 열면 가장 먼저 알아야 할 것은 현재 위치를 확인하고, 파일을 탐색하는 방법입니다. 이 세 가지 명령어만 익혀도 업무 효율이 눈에 띄게 달라집니다.

### 1. 현재 위치 확인: pwd

`pwd`는 "print working directory"의 약자로, 현재 작업 중인 폴더 경로를 보여줍니다. 터미널을 열면 기본적으로 사용자 홈 폴더(`/Users/사용자이름`)에 위치하게 됩니다. 이 명령어를 입력하면 자신이 어디에 있는지 정확히 알 수 있어 길을 잃지 않습니다.

### 2. 폴더 내용 보기: ls

`ls`는 현재 폴더 안에 어떤 파일과 폴더가 있는지 목록으로 출력합니다. `ls -la` 옵션을 추가하면 숨김 파일까지 자세히 볼 수 있어, 시스템 파일을 확인하거나 불필요한 캐시를 정리할 때 유용합니다.

### 3. 폴더 이동하기: cd

`cd`는 디렉토리를 변경하는 명령어입니다. `cd Documents`처럼 입력하면 Documents 폴더로 이동하고, `cd ..`를 입력하면 상위 폴더로 돌아갑니다. 자주 사용하는 폴더로 빠르게 이동하는 습관을 들이면 터미널 작업 속도가 2배 이상 빨라집니다.

## 터미널 자동화로 하루 30분 절약하기

기본 명령어를 익혔다면, 이제 반복 작업을 자동화하는 단계로 넘어가야 합니다. 터미널을 활용하면 매일 하는 파일 백업, 이름 변경, 불필요한 파일 삭제를 명령어 하나로 처리할 수 있습니다.

### 파일 일괄 이름 변경

여러 파일의 이름을 한 번에 바꾸고 싶다면 `rename` 명령어를 사용하세요. 예를 들어, 모든 `.txt` 파일을 `.md`로 바꾸려면 `rename 's/\.txt/\.md/' *.txt`라고 입력하면 됩니다. 이 방법은 사진 파일 정리나 프로젝트 문서 관리에서 특히 유용합니다.

### 폴더 내 중복 파일 찾기

`fdupes` 같은 유틸리티를 설치하면 폴더 안에 있는 중복 파일을 자동으로 찾아서 삭제할 수 있습니다. 설치 후 `fdupes -r .` 명령어를 실행하면 현재 폴더 내 모든 중복 파일이 리스트로 출력되므로, 수동으로 하나하나 확인할 필요가 없습니다.

### 자주 쓰는 명령어를 별칭(alias)으로 저장

매번 긴 명령어를 입력하는 대신, `.zshrc` 파일에 별칭을 등록해 두세요. 예를 들어 `alias ll='ls -la'`를 추가하면 `ll`만 입력해도 전체 목록을 볼 수 있습니다. 이 작은 팁 하나로 터미널 명령어 입력 시간을 50% 이상 줄일 수 있습니다.

### 히스토리 검색으로 명령어 재사용

`Ctrl + R`을 누르면 이전에 입력했던 명령어를 검색할 수 있습니다. 예를 들어 복잡한 경로의 파일을 복사한 명령어를 다시 찾고 싶다면, `Ctrl + R`을 누르고 키워드를 입력하면 바로 불러올 수 있습니다. 이 기능을 모르면 똑같은 명령어를 여러 번 타이핑해야 하지만, 익숙해지면 작업 효율이 비약적으로 상승합니다.

### 파이프라인으로 여러 명령어 연결

파이프(`|`)를 사용하면 앞 명령어의 결과를 바로 다음 명령어로 전달할 수 있습니다. 예를 들어 `ls | grep ".pdf"`를 입력하면 현재 폴더에서 PDF 파일만 골라서 보여줍니다. 여기에 `wc -l`을 추가하면 PDF 파일 개수까지 한 번에 확인할 수 있습니다.

### 터미널 멀티태스킹: tmux 활용

`tmux`를 설치하면 하나의 터미널 창에서 여러 세션을 동시에 실행할 수 있습니다. 화면을 분할해 왼쪽에서는 로그를 모니터링하고, 오른쪽에서는 명령어를 입력하는 식으로 작업하면 번거로운 창 전환이 필요 없어집니다.

### 작업 중단과 재개: Ctrl+Z와 fg

긴 작업을 실행 중인데 급히 다른 명령어를 입력해야 한다면 `Ctrl+Z`로 일시 중단하고, `fg` 명령어로 다시 재개하세요. 이 방법을 모르면 새 터미널 창을 열거나 기존 작업을 강제 종료해야 하지만, 이 팁 하나로 작업 흐름을 끊지 않고 유지할 수 있습니다.
