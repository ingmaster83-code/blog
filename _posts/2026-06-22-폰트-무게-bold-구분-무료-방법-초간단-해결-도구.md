## 같은 글자, 다른 느낌… 폰트 무게가 당신의 문서를 망친다

직장인이라면 누구나 한 번쯤 겪어봤을 고민입니다. PPT 제목에 강조를 주려고 Bold를 적용했는데, 옆에 있는 동료의 폰트는 더 진하고, 제 폰트는 얇아 보입니다. 혹은 학생이 과제를 할 때, 교수님이 지정한 폰트인데 Bold가 제대로 적용되었는지 확신이 서지 않는 순간. 이럴 때 필요한 것이 바로 **폰트 무게 bold 구분 무료 방법**입니다. 폰트마다 동일한 Bold라고 해도 실제 굵기가 다르기 때문에, 확인 없이 사용하면 문서의 통일성이 무너지기 쉽습니다.

어도비 일러스트레이터나 포토샵 같은 전문 프로그램을 켜서 확인할 수도 있지만, 매번 프로그램을 실행하는 것은 번거롭습니다. 게다가 해당 프로그램이 설치되지 않은 환경이라면 더 난감하죠. 이런 문제를 해결하기 위해 제가 실제로 사용하고 있는 꿀팁을 공개합니다.

## 폰트 굵기 확인 방법, 이제 FontKit 하나면 끝

복잡한 설치나 회원가입 없이, 웹 브라우저만 있으면 바로 사용할 수 있는 도구가 있습니다. 바로 [FontKit](https://fontkit.wooahouse.com)입니다. 이 도구는 폰트 파일을 업로드하면 해당 폰트의 모든 속성을 한눈에 보여줍니다. 특히 **폰트 무게(Font Weight)** 값을 정확히 분석해주기 때문에, 일반 Bold(700)인지, ExtraBold(800)인지, 아니면 Light(300)인지 즉시 구분할 수 있습니다.

저도 처음에는 반신반의하며 사용해봤는데, 결과가 너무 정확해서 깜짝 놀랐습니다. 게다가 **무료**이기 때문에 예산 걱정도 없습니다. 지금부터 FontKit을 사용하여 폰트 무게를 구분하는 구체적인 단계를 알려드리겠습니다.

### 1단계: FontKit 웹사이트 접속

가장 먼저 해야 할 일은 [FontKit 웹사이트](https://fontkit.wooahouse.com)에 접속하는 것입니다. 별도의 다운로드나 설치 과정이 전혀 없습니다. 크롬, 사파리, 엣지 등 어떤 브라우저든 상관없이 바로 사용할 수 있습니다.

### 2단계: 확인하려는 폰트 파일 업로드

사이트에 접속하면 중앙에 파일 업로드 영역이 보입니다. 여기에 확인하고 싶은 폰트 파일(.ttf, .otf, .woff 등)을 드래그 앤 드롭하거나, 영역을 클릭하여 파일을 선택하면 됩니다. 여러 개의 폰트를 동시에 업로드하여 비교할 수도 있어서, **폰트 굵기 비교** 작업에 매우 효율적입니다.

### 3단계: Font Weight 값 확인

파일이 업로드되면 잠시 후 FontKit이 폰트의 모든 메타데이터를 분석하여 보여줍니다. **글꼴 무게 구분**을 위해 가장 중요한 항목은 'Font Weight' 값입니다. 여기에 표시되는 숫자가 곧 폰트의 굵기입니다.

- **Thin / Hairline (100)**
- **ExtraLight / UltraLight (200)**
- **Light (300)**
- **Normal / Regular (400)**
- **Medium (500)**
- **SemiBold / DemiBold (600)**
- **Bold (700)**
- **ExtraBold / UltraBold (800)**
- **Black / Heavy (900)**

만약 'Bold'라고 이름이 붙은 폰트인데 Font Weight 값이 700이 아니라 600(SemiBold)으로 표시된다면, 이 폰트는 일반적인 Bold보다 얇은 것입니다. 반대로 800으로 표시된다면 더 두꺼운 폰트라는 뜻이죠. 이렇게 **폰트 속성 보기**를 통해 정확한 굵기를 알 수 있습니다.

### 4단계: 추가 폰트 정보 활용하기

FontKit은 무게뿐만 아니라 **웹폰트 두께** 설정에 필요한 CSS 정보도 함께 제공합니다. 'font-family', 'font-style', 'font-weight' 등의 값을 복사하여 웹사이트에 바로 적용할 수 있습니다. 디자이너나 웹 개발자라면 이 기능이 특히 유용할 것입니다.

## 사용 시 주의사항

FontKit은 매우 직관적이지만, 몇 가지 주의할 점이 있습니다.

1. **파일 형식 확인**: 대부분의 폰트 파일을 지원하지만, 아주 오래된 PostScript Type1 포맷 등 일부 레거시 폰트는 지원하지 않을 수 있습니다.
2. **인터넷 연결**: 모든 처리가 로컬에서 이루어지지만, 초기 페이지 로딩을 위해 인터넷 연결이 필요합니다.
3. **폰트 라이선스**: 폰트 파일 자체는 본인이 합법적으로 소유한 폰트만 업로드하는 것이 좋습니다. 배포 권한이 없는 폰트를 무단으로 업로드하는 것은 삼가야 합니다.
