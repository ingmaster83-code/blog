## 포트 확인 온라인 무료 사이트 WooaDev 소개

제가 최근에 알게 된 [WooaDev 무료로 사용하기](https://wooadev.wooahouse.com)는 별도의 프로그램 설치가 전혀 필요 없는 웹 기반 포트 스캐너입니다. 네트워크 포트 검사가 필요할 때마다 CMD나 PuTTY를 켤 필요 없이 브라우저만 열면 끝입니다. 특히 외부에서 급하게 서버 포트 오픈 여부를 확인해야 할 때 노트북 하나로 바로 처리할 수 있어서 정말 편리합니다.

이 사이트의 가장 큰 장점은 인터페이스가 직관적이라는 점입니다. 포트 체크를 위해 복잡한 옵션을 설정할 필요 없이 IP 주소와 포트 번호만 입력하면 됩니다. 무료 온라인 포트 테스트 도구 중에서 이렇게 심플한 건 처음 봤습니다.

## 포트 열림 확인하는 단계별 사용법

실제로 제가 서버 포트가 제대로 열렸는지 테스트해본 과정을 그대로 알려드리겠습니다. 누구나 따라 하면 30초 안에 끝납니다.

### 1단계: WooaDev 사이트 접속하기

먼저 [WooaDev](https://wooadev.wooahouse.com)에 접속합니다. 메인 화면에는 "IP 주소 또는 도메인" 입력창과 "포트 번호" 입력창 두 개만 보입니다. 복잡한 광고나 불필요한 버튼이 없어서 처음 보는 사람도 당황하지 않습니다.

### 2단계: 대상 정보 입력하기

IP 주소 칸에는 검사하고 싶은 서버의 공인 IP를 입력합니다. 만약 도메인을 알고 있다면 example.com 같은 형태로 입력해도 자동으로 IP를 찾아서 검사해줍니다. 포트 번호 칸에는 80, 443, 3306 등 확인하고 싶은 포트를 입력합니다. 웹 서버 포트 확인을 위해 80번 포트를 넣어봤는데 매우 반응이 빨랐습니다.

### 3단계: 검사 시작 및 결과 확인

입력을 마친 후 "포트 검사" 버튼을 클릭하면 3~5초 안에 결과가 나타납니다. "열림"이라는 초록색 배지가 뜨면 포트가 정상적으로 열려 있는 상태입니다. 반대로 "닫힘"이 뜨면 방화벽이나 서비스 설정을 의심해봐야 합니다. 저는 AWS 보안 그룹에서 22번 포트를 막아놓은 상태에서 테스트해봤는데 정확하게 "닫힘"으로 표시되더군요.

## 사용 시 주의사항

이 도구는 TCP 포트 스캔 방식으로 동작합니다. UDP 포트는 지원하지 않으므로 DNS나 DHCP 같은 UDP 기반 서비스를 테스트할 때는 다른 방법을 써야 합니다. 또한 대상 서버에 방화벽이 ICMP를 차단하고 있더라도 TCP 포트 검사 자체에는 문제가 없습니다.

개인 네트워크 내부의 포트를 확인하려면 해당 기기가 외부에서 접근 가능한 상태여야 합니다. 공유기 아래 있는 내부 IP(192.168.x.x)는 이 사이트에서 직접 검사할 수 없으므로, 공유기의 포트 포워딩 설정을 먼저 확인하세요.
