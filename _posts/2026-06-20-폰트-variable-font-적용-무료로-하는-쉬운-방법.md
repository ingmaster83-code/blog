웹사이트 속도를 높이고 싶어서 구글 폰트에서 variable font를 찾아보신 적 있나요? 저도 최근에 블로그를 리뉴얼하면서 "폰트 variable font 적용" 방법을 찾고 있었는데, 생각보다 까다롭더라고요. 직접 CSS를 수정하고, 여러 파일을 다운로드 받아야 하는 번거로움에 한참 헤맸습니다.

그러던 중 우연히 발견한 무료 온라인 도구가 해결사가 되어줬습니다. 더 이상 복잡한 코드를 만질 필요도 없고, 돈을 지불할 필요도 없습니다. 이 글에서는 제가 실제로 사용해본 **FontKit**을 통해 **폰트 variable font 적용을 무료**로 끝내는 방법을 단계별로 알려드리려고 합니다.

## Variable font 적용이 필요한 이유와 해결 도구

기존에는 웹사이트에서 굵은 글씨, 얇은 글씨, 기울임꼴 등을 표현하려면 각각의 개별 폰트 파일(예: NotoSansKR-Bold.woff, NotoSansKR-Light.woff)을 따로 불러와야 했습니다. 이렇게 하면 페이지 로딩 속도가 느려지고, 파일 관리도 어렵습니다.

반면 **variable font(가변 글꼴)** 는 하나의 파일로 굵기, 너비, 기울기 등 모든 속성을 조절할 수 있습니다. 즉, 서버 요청 횟수를 줄여 속도를 높이고, 디자인의 자유도는 높이는 일석이조의 효과를 누릴 수 있습니다.

하지만 문제는 일반 폰트를 variable font로 변환하는 과정이 일반 사용자에게는 너무 어렵다는 점입니다. 바로 이 지점에서 **FontKit**이 등장합니다. 별도의 설치나 회원가입 없이 웹 브라우저만 있으면 누구나 즉시 사용할 수 있는 **무료 폰트 변환 도구**입니다.

## FontKit으로 폰트 variable font 적용하는 방법

### 1단계: 변환할 폰트 파일 준비하기

먼저 변환하고 싶은 TTF, OTF, WOFF, WOFF2 형식의 폰트 파일을 컴퓨터에 준비합니다. 예를 들어, 상업용으로 무료인 '눈누' 폰트나 '네이버 나눔스퀘어' 같은 인기 폰트를 추천드립니다. 파일 하나만 있으면 되며, 여러 개의 굵기(예: Light, Regular, Bold)가 포함된 폰트 패밀리 파일이면 더욱 좋습니다.

### 2단계: FontKit 사이트에 접속해서 파일 업로드하기

위에 링크된 [FontKit](https://fontkit.wooahouse.com) 사이트에 접속하면 메인 화면에 파일을 끌어다 놓는 영역이 보입니다. 여기에 준비한 폰트 파일을 드래그 앤 드롭하거나, '파일 선택' 버튼을 클릭해서 업로드하면 됩니다. 저는 개인적으로 드래그 앤 드롭이 가장 편리하더군요.

### 3단계: 웹 폰트 형식 설정 및 변환 완료

파일이 업로드되면 사이트가 자동으로 폰트를 분석합니다. 이후 'WOFF2로 변환' 또는 'Variable Font로 변환' 같은 옵션이 나타납니다. 여기서 **Variable Font** 옵션을 선택하고 '변환 시작' 버튼을 누르면 몇 초 안에 변환이 완료됩니다. **설치 불필요**하고 추가 설정도 전혀 없어서 정말 간편합니다.

### 4단계: 변환된 파일 다운로드 및 웹사이트에 적용

변환이 끝나면 '다운로드' 버튼이 활성화됩니다. 변환된 .woff2 파일을 다운로드한 후, 자신의 웹사이트 CSS 폴더에 업로드하고 아래와 같은 코드로 불러오면 됩니다.

```css
@font-face {
    font-family: 'MyVariableFont';
    src: url('MyVariableFont.woff2') format('woff2');
    font-weight: 1 999; /* 모든 굵기 허용 */
}
```

이제 CSS에서 `font-weight: 350;` 같은 세밀한 값을 지정하면 완벽하게 **variable font 적용**이 완료됩니다. 페이지 로딩 속도가 눈에 띄게 개선된 것을 체감할 수 있을 거예요.

## 사용 시 주의사항 및 꿀팁

가장 중요한 주의사항은 **파일 크기**입니다. Variable font는 하나의 파일로 다양한 스타일을 담기 때문에 원본 TTF 파일보다 크기가 더 커질 수 있습니다. 하지만 WOFF2 형식으로 압축되므로 일반적으로 여러 개의 개별 폰트를 로딩하는 것보다는 훨씬 가볍습니다.

또 하나, 변환 후 모든 폰트가 완벽하게 '가변' 속성을 가지는 것은 아닙니다. 원본 폰트 자체에 다양한 굵기 데이터가 포함되어 있어야 합니다. 만약 변환 후에도 굵기가 조절되지 않는다면, 원본 폰트가 단일 굵기만 지원하는 폰트였기 때문입니다. 이 경우 굵기가 다양한 폰트 패밀리 파일을 찾아 다시 시도해보세요.

마지막으로, **모바일 호환성**을 꼭 확인하세요. 최신 브라우저(크롬, 사파리, 파이어폭스 등)는 모두 variable font를 지원하지만, 아주 오래된 브라우저에서는 적용되지 않을 수 있습니다. 필요하다면 CSS에 `@supports` 구문을 추가해서 대체 폰트를 지정해주는 것이 안전합니다.
