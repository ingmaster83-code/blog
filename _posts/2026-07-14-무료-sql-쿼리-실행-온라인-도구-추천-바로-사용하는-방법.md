### 핵심 기능과 사용 방법

이 도구는 단순하면서도 실용적인 기능들을 제공합니다. 주요 기능을 하나씩 살펴보겠습니다.

#### 기본 SQL 실행 환경

에디터에 SQL 문을 입력하고 실행 버튼을 누르면 결과 테이블이 즉시 표시됩니다. SELECT, INSERT, UPDATE, DELETE 같은 기본 DML과 CREATE TABLE 같은 DDL을 모두 지원합니다. 복잡한 JOIN이나 서브쿼리도 문제없이 수행되어 **SQL 쿼리 연습**에 최적화되어 있습니다.

#### 샘플 데이터베이스 활용

처음 접속하면 기본 샘플 테이블이 제공됩니다. 사용자, 주문, 제품 등 실제 비즈니스 시나리오와 유사한 데이터로 **데이터베이스 쿼리 테스트**를 바로 시작할 수 있습니다. 이 기능은 SQL을 배우는 초보자에게 특히 유용합니다.

#### CSV 파일 업로드 및 내보내기

자신의 데이터를 테스트해야 한다면 CSV 파일을 업로드하여 테이블로 변환할 수 있습니다. 반대로 실행 결과를 CSV로 다운로드하는 기능도 지원합니다. 다만 대용량 파일(10MB 이상)에서는 처리 속도가 다소 느려질 수 있으므로, 간단한 데이터 세트에 사용하는 것을 권장합니다.

### 실제 사용 후기: 장점과 단점

실제로 2주간 다양한 쿼리를 테스트하며 느낀 점을 정리했습니다.

#### 장점

- **즉시 실행 가능**: 설치, 설정, 회원가입이 전혀 필요 없습니다. URL만 알면 누구나 30초 안에 SQL을 실행할 수 있습니다.
- **개인정보 보호**: 모든 쿼리는 브라우저 내에서만 처리되므로 민감한 데이터도 안심하고 사용할 수 있습니다. 이 점은 **SQL 테스트 도구**를 선택할 때 가장 중요한 요소 중 하나입니다.

- **가벼운 인터페이스**: 불필요한 기능이 없어 직관적입니다. 쿼리 입력과 결과 확인에 집중할 수 있습니다.

#### 단점

- **고급 기능 부족**: 인덱스 튜닝, 실행 계획 분석, 프로파일링 같은 고급 기능은 제공되지 않습니다. **SQL 쿼리 실행 온라인 도구**로서 기본에 충실하지만, DBA 레벨의 작업에는 한계가 있습니다.
- **데이터 지속성**: 브라우저를 닫으면 생성한 테이블과 데이터가 사라집니다. 장기 프로젝트보다는 일회성 테스트나 학습용으로 적합합니다.
- **SQLite 한계**: SQLite 엔진을 사용하므로 MySQL, PostgreSQL 전용 문법(예: `LIMIT` 대신 `TOP`, 윈도우 함수 일부)은 지원되지 않을 수 있습니다.
