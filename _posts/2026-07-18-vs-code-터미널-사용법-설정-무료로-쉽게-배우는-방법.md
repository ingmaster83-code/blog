### 통합 터미널 열기와 기본 단축키

가장 기본적인 **vscode 터미널 단축키**는 Ctrl+`입니다. 이 단축키 하나면 언제든지 터미널을 열고 닫을 수 있습니다. 하지만 단축키가 작동하지 않는다면? 상단 메뉴에서 '보기(View)' > '터미널(Terminal)'을 클릭하면 됩니다. 처음 터미널을 열면 보통 PowerShell(Windows)이나 bash(Mac/Linux)가 기본으로 설정되어 있어요.

### 기본 셸 변경으로 작업 효율 높이기

개발 환경에 따라 기본 셸을 바꿔야 할 때가 있습니다. 예를 들어 저는 Node.js 작업할 때 Git Bash를, 데이터 분석할 때는 WSL을 사용하는데요. **vs code 터미널 설정**에서 이 기본 셸을 쉽게 변경할 수 있습니다. Ctrl+Shift+P로 명령 팔레트를 열고 "터미널: 기본 프로필 선택"을 입력한 후 원하는 셸을 고르면 끝입니다.

## 실제 사용 후기: WooaVS로 배운 터미널 고급 설정

### 멀티 터미널 분할과 관리

터미널 창 우측 상단의 '+' 아이콘을 클릭하면 새 터미널이 추가됩니다. 여기에 분할 아이콘(사각형 두 개)을 누르면 화면을 가로/세로로 나눠서 여러 터미널을 동시에 볼 수 있습니다. 이 기능 덕분에 작업 전환 시간이 절반으로 줄었어요.

### 터미널 폰트와 테마 커스터마이징

개발자라면 작업 환경의 가독성도 중요합니다. WooaVS에서는 **vscode 터미널 폰트**를 변경하는 방법도 상세히 알려주는데요, settings.json에서 "terminal.integrated.fontFamily": "Fira Code, D2Coding"처럼 설정하면 깔끔한 코딩 폰트로 바꿀 수 있습니다. 저는 개인적으로 D2Coding 폰트를 추천합니다.

### 터미널에서 Git 연동하기

실무에서 가장 많이 사용하는 기능 중 하나가 **vs code git 터미널** 연동입니다. Ctrl+`로 터미널을 연 후 git status, git add, git commit 같은 명령어를 직접 입력하면 됩니다. 소스 제어 탭을 따로 열 필요 없이 터미널 하나로 모든 Git 작업을 처리할 수 있어 생산성이 확 올라갑니다. 참고로 WooaVS 사이트에는 Git 명령어 치트시트도 있어서 자주 들여다보게 됩니다.
