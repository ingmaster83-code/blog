# 폰트 CSS @font-face 코드 무료 생성, 직장인과 학생을 위한 꿀팁

회사에서 보고서를 작성하거나 대학 과제로 PPT를 만들 때, 예쁜 폰트를 찾아서 적용하려다 낭패를 본 적이 있을 겁니다. 디자인 파일에는 잘 나오는데 막상 웹에 업로드하거나 다른 사람에게 공유하면 폰트가 깨져서 기본 글꼴로 보이는 경험, 다들 한 번쯤 있으시죠? 이런 문제를 해결하려면 **폰트 CSS @font-face 코드 무료**로 생성해서 웹사이트에 직접 적용해야 합니다. 하지만 CSS를 잘 모르는 직장인이나 학생에게는 이 과정이 마치 외계어처럼 느껴질 수 있습니다.

이 문제를 해결해 줄 완벽한 도구가 있습니다. 바로 **FontKit**입니다. FontKit은 TTF나 OTF 파일을 업로드하면 자동으로 웹폰트 형식(WOFF2, WOFF 등)으로 변환하고, 바로 사용할 수 있는 CSS @font-face 코드를 생성해 줍니다. 설치나 회원가입 없이 브라우저에서 무료로 사용할 수 있어 정말 편리합니다.

## FontKit으로 @font-face 웹폰트 적용하는 방법

이제 실제로 어떻게 사용하는지 단계별로 알아보겠습니다. 어렵게 생각할 필요 없이 아래 순서대로 따라오시면 됩니다.

### 1단계: FontKit 접속 및 폰트 파일 업로드

먼저 [FontKit 사이트](https://fontkit.wooahouse.com)에 접속합니다. 화면 중앙에 있는 큰 업로드 버튼을 클릭하거나, 사용하려는 폰트 파일을 드래그 앤 드롭으로 넣어주세요. 대부분의 일반 폰트 파일(TTF, OTF, EOT 등)을 지원하므로 파일 형식 걱정은 하지 않으셔도 됩니다.

### 2단계: CSS 코드와 웹폰트 파일 다운로드

파일을 업로드하면 몇 초 안에 변환 작업이 완료됩니다. 그러면 화면에 두 가지 결과물이 나타납니다. 첫째는 **웹폰트 변환 파일(주로 WOFF2 형식)** 이고, 둘째는 **완성된 @font-face CSS 코드**입니다. 여기서 중요한 점은 CSS 코드를 바로 복사해서 자신의 프로젝트에 붙여넣고, 변환된 폰트 파일도 함께 다운로드받아 서버에 업로드해야 한다는 것입니다.

### 3단계: 실제 프로젝트에 CSS 코드 적용하기

복사한 CSS 코드를 HTML 파일 안에 `<style>` 태그로 넣거나, 별도의 CSS 파일에 붙여넣습니다. 그런 다음 변환된 폰트 파일을 웹 서버의 적절한 폴더(예: `/fonts/`)에 업로드하고, CSS 코드 안의 `src: url()` 경로를 해당 파일 위치에 맞게 수정해 주면 됩니다. 이제 CSS에서 `font-family` 속성으로 해당 폰트를 자유롭게 사용할 수 있습니다.

## FontKit 사용 시 주의사항

- **저작권 확인 필수**: FontKit은 기술적으로 폰트를 변환해 주지만, 사용자가 업로드한 폰트의 저작권까지 책임지지 않습니다. 상업용 폰트나 유료 폰트를 웹에 사용할 때는 반드시 라이선스를 확인하세요.
- **파일 경로 정확히 입력**: 생성된 CSS 코드의 경로는 상대 경로로 표시되므로, 실제 서버 구조에 맞게 `url("../fonts/myfont.woff2")` 형태로 수정해야 합니다.
- **브라우저 호환성**: WOFF2 형식은 대부분의 최신 브라우저에서 지원되지만, 구형 브라우저를 고려한다면 WOFF나 TTF 형식을 추가로 포함하는 것이 좋습니다.
