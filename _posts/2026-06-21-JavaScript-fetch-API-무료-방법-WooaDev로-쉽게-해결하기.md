## WooaDev로 fetch API 요청하는 단계별 가이드

### ### 1. 도구 접속 및 인터페이스 확인

먼저 브라우저에서 WooaDev 사이트에 접속합니다. 메인 화면에는 URL 입력창, HTTP 메서드 선택 드롭다운, 헤더 입력 영역, 그리고 요청 본문(body) 입력란이 직관적으로 배치되어 있습니다. 복잡한 설명 없이 바로 사용할 수 있는 심플한 UI가 특징입니다.

### ### 2. HTTP 메서드와 URL 설정

API 호출을 위해 원하는 **HTTP 요청 방식**을 선택합니다. GET, POST, PUT, DELETE, PATCH 등 모든 주요 메서드를 지원합니다. 이후 API 엔드포인트 URL을 입력창에 붙여넣기 합니다. 이 과정에서 URL 유효성 검사가 자동으로 이루어져 오타를 방지해줍니다.

### ### 3. 헤더와 바디 구성

REST API 호출 시 필요한 헤더(Authorization, Content-Type 등)를 추가합니다. POST나 PUT 요청의 경우 JSON, Form-data 등 다양한 형식으로 요청 본문을 작성할 수 있습니다. **실시간 구문 강조** 기능이 적용되어 JSON 형식 오류를 즉시 확인할 수 있어 매우 편리합니다.

### ### 4. 요청 전송 및 응답 확인

Send 버튼을 클릭하면 즉시 요청이 전송되고, 응답 상태 코드, 헤더, 본문이 깔끔하게 표시됩니다. 응답 데이터는 자동으로 포맷팅되어 가독성이 높으며, **코드 복사 버튼**을 통해 JavaScript fetch 코드를 한 번에 복사할 수 있습니다. 이 기능은 특히 반복적인 API 테스트 작업에서 시간을 크게 절약해줍니다.
