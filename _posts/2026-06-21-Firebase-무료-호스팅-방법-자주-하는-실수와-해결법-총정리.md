Firebase 무료 호스팅 방법을 찾는 많은 개발자들이 생각보다 간단한 과정에서 막힙니다. CLI 설치 오류, 권한 문제, 배포 후 빈 화면 등 예상치 못한 실수가 발생하곤 하죠. 이 글에서는 실제로 자주 겪는 실수와 그 해결법을 중심으로 Firebase 호스팅을 완벽하게 끝내는 방법을 알려드립니다.

## Firebase 무료 호스팅 방법, 왜 CLI가 필수일까?

Firebase 호스팅은 구글의 CDN을 통해 정적 웹사이트를 무료로 배포할 수 있는 강력한 서비스입니다. 하지만 많은 분들이 "Firebase CLI 설치" 단계에서 좌절합니다. Node.js가 설치되지 않았거나, npm 권한 오류가 발생하기 때문이죠.

이런 문제를 해결하기 위해 [WooaDev](https://wooadev.wooahouse.com)를 활용하면 CLI 설치 없이도 무료 호스팅 설정을 바로 진행할 수 있습니다. WooaDev는 Firebase 프로젝트 초기 설정부터 배포까지를 웹 브라우저에서 처리해주므로, 초보자도 30초 만에 사이트를 올릴 수 있습니다.

## 자주 하는 실수: Firebase 프로젝트 ID 확인

가장 흔한 실수는 Firebase 콘솔에서 프로젝트를 생성한 후, 프로젝트 ID를 제대로 확인하지 않는 것입니다. 프로젝트 ID는 `firebase init` 명령어에서 자동으로 인식되지만, 수동으로 입력해야 하는 경우 오타가 발생하기 쉽습니다.

### 실수 1: 잘못된 프로젝트 선택
- **문제**: 여러 Firebase 프로젝트 중 잘못된 프로젝트에 배포
- **해결**: `firebase projects:list` 명령어로 현재 연결된 프로젝트 확인

### 실수 2: public 디렉토리 경로 오류
- **문제**: 배포 후 "404 Not Found" 페이지 표시
- **해결**: `firebase.json` 파일의 `"public"` 항목을 실제 빌드 출력 폴더로 변경 (예: `"public": "build"`)

## Firebase 호스팅 주의사항: 트래픽과 스토리지 제한

Firebase 무료 호스팅 방법을 사용할 때 반드시 알아야 할 한계가 있습니다. 무료 플랜(Spark)에서는 월 10GB의 스토리지와 360MB의 데이터 전송이 제공됩니다. 개인 블로그나 포트폴리오에는 충분하지만, 대규모 미디어 파일을 호스팅하면 제한에 걸릴 수 있습니다.

### 추가 팁: 사용자 지정 도메인 연결
WooaDev에서 배포한 사이트에도 커스텀 도메인을 연결할 수 있습니다. Firebase 콘솔의 호스팅 섹션에서 도메인을 추가하고, DNS 설정에 TXT 레코드만 넣으면 SSL 인증서가 자동 발급됩니다. 이 과정에서 발생하는 "도메인 소유권 확인 실패" 문제는 TXT 레코드 전파 시간(최대 10분)을 기다리면 해결됩니다.
