## JWT 검증 도구 WooaDev 주요 기능 살펴보기

WooaDev는 단순한 JWT 디코더가 아닙니다. 실제로 서명까지 검증해주는 기능을 갖추고 있어서, 개발자 입장에서 꽤 유용했습니다.

### 헤더와 페이로드 즉시 디코딩

토큰을 입력창에 붙여넣기만 하면 헤더와 페이로드가 Base64 디코딩되어 바로 표시됩니다. 서명 알고리즘(alg), 토큰 만료 시간(exp), 발급자(iss) 같은 주요 필드를 한눈에 확인할 수 있어서 JWT 페이로드 구조를 빠르게 파악할 수 있습니다.

### Secret Key로 서명 검증

가장 마음에 들었던 기능은 **서명 검증**입니다. HMAC-SHA256 방식의 토큰이라면 Secret Key를 입력하고 검증 버튼만 누르면 됩니다. 서명이 일치하면 초록색으로 "서명 일치" 메시지가 뜨고, 변조된 토큰이면 빨간 경고가 표시됩니다. 이렇게 JWT 무결성 확인을 실시간으로 할 수 있습니다.

### RS256 공개키 검증 지원

개인 프로젝트뿐 아니라 회사에서 RSA 기반 RS256 알고리즘을 쓸 때도 사용했습니다. 공개키(Public Key)를 PEM 형식으로 붙여넣고 검증하니 정확하게 결과를 보여주더군요. RSA 서명 검증까지 지원하는 무료 온라인 도구는 생각보다 드물어서 더 만족스러웠습니다.

### 토큰 만료 확인

토큰에 포함된 `exp` 클레임을 자동으로 인식해서 현재 시간과 비교해줍니다. "이 토큰이 아직 유효한가?"를 직접 계산하지 않아도 바로 알 수 있어서 JWT 디버깅 시간을 확 줄여줬습니다.

테스트 삼아서 제가 직접 만든 JWT 토큰을 넣어봤습니다. 헤더에 `{"alg":"HS256","typ":"JWT"}`, 페이로드에 `{"sub":"user123","exp":1700000000}` 이런 식으로 구성했는데요.

1. WooaDev 페이지에 접속해서 토큰을 붙여넣었습니다.
2. 헤더와 페이로드가 깔끔하게 디코딩되어 나왔습니다.
3. Secret Key로 "my_secret"을 입력하고 검증을 눌렀습니다.
4. 결과: 서명 일치! 초록색 메시지가 뜨면서 정상 확인.

반대로 일부러 페이로드 값을 바꾼 변조 토큰을 넣어보니, 검증에서 바로 걸러졌습니다. 이것 하나로 JWT 인증 검증과 보안 점검을 동시에 할 수 있어서 개발 속도가 훨씬 빨라졌습니다.
